Frontend-Backend Debugging Anleitung

Verbindungsprobleme zwischen Frontend und Backend mit Browser DevTools analysieren

1. DevTools offnen

BrowserTastenkombination
Chrome/EdgeF12 oder Strg + Shift + I
FirefoxF12 oder Strg + Shift + I
SafariCmd + Option + I (erst in Einstellungen aktivieren)

2. Network Tab - API Requests analysieren

Schritt-fur-Schritt:

  1. Network Tab offnen (oben in DevTools)
  2. Filter setzen: Klick auf Fetch/XHR (zeigt nur API-Calls)
  3. Seite neu laden oder Aktion ausfuhren die fehlschlagt
  4. Roten Request suchen = Fehler

Status-Codes:

3. CORS-Fehler erkennen und beheben

So erkennst du CORS-Probleme:

  1. Console Tab offnen (neben Network)
  2. Suche nach roter Meldung:
"Access to fetch at 'https://api.example.com' from origin
'http://localhost:3000' has been blocked by CORS policy"

Was ist CORS?

Browser blockieren Requests an andere Domains aus Sicherheitsgrunden.

Frontend: http://localhost:3000
Backend:  http://localhost:8000    ← Andere Port = Andere "Origin"
                                   ← Browser blockiert!

Losung im Backend:

Node.js/Express:

const cors = require('cors');
app.use(cors({
  origin: 'http://localhost:3000',  // Frontend URL
  credentials: true
}));

Django:

# settings.py
CORS_ALLOWED_ORIGINS = [
    "http://localhost:3000",
]

FastAPI:

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

5. Haufige Probleme und Losungen

Problem: Request wird gar nicht gesendet

Symptom: Kein Request im Network Tab sichtbar

Ursachen:

  • JavaScript-Fehler vorher → Console Tab prufen
  • Event Handler nicht verbunden
  • Falscher Button/Form

Problem: "net::ERR_CONNECTION_REFUSED"

Symptom: Request failed, keine Response

Ursachen:

  • Backend lauft nicht
  • Falsche URL/Port
  • Firewall blockiert

Prufen:

# Lauft der Backend-Server?
curl http://localhost:8000/health

# Oder im Browser direkt aufrufen

Problem: Request hangt (Pending)

Symptom: Request bleibt gelb, kein Status

Ursachen:

  • Backend antwortet nicht (Timeout)
  • Endlosschleife im Backend
  • Datenbankverbindung blockiert

Problem: Falscher Content-Type

Symptom: 400 Bad Request, obwohl Daten korrekt

Ursache: Server erwartet anderes Format

// Falsch:
fetch('/api/data', {
  method: 'POST',
  body: JSON.stringify(data)
})

// Richtig:
fetch('/api/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'  // ← Das fehlt oft!
  },
  body: JSON.stringify(data)
})

7. Debugging Checkliste

Wenn API-Calls nicht funktionieren, prufe in dieser Reihenfolge:

  • 1. Console Tab: JavaScript-Fehler vorhanden?
  • 2. Network Tab: Request wurde gesendet?
  • 3. Status Code: Was antwortet der Server?
  • 4. CORS: Rote Meldung in Console?
  • 5. URL korrekt? Tippfehler? Trailing Slash?
  • 6. Headers: Authorization Token dabei?
  • 7. Payload: Richtiges Format (JSON)?
  • 8. Backend lauft? Prozess checken
  • 9. Backend Logs: Fehler auf Server-Seite?

8. Nutzliche DevTools Features

Checkbox oben im Network Tab - Erzwingt frische Requests

Throttling
